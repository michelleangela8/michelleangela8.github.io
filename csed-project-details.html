CSED332 Software Design Methods

Project Overview

IntelliJ IDEAâ€™s built-in test coverage visualization presents results in large, text-heavy tables that are difficult to interpret at a glance. Additionally, the in-line coverage display is bulky and does not efficiently convey essential information. Developers need a more intuitive, graphical representation of test coverage to improve debugging and code analysis.

Solution

Our team developed an IntelliJ IDEA plugin that enhances test coverage visualization by:

  • Condition-wise Test Coverage Display: Shows which test cases cover specific conditions.
  • Graphical Representation : Generates visual graphs for test coverage results.
  • Popup Window UI: Allows developers to view detailed coverage graphs in a separate window.

Design Considerations

  • Seamless Integration: Built as an IntelliJ plugin using the Code Coverage for Java API.
  • User-Friendly Visualization: Implements graph-based representations for better readability.
  • Performance Optimization: Efficiently processes coverage data without slowing down IntelliJ.

Impact

This test coverage visualization plugin enhances software development workflows by improving readability and efficiency in analyzing test results. The graphical insights help developers identify untested conditions more quickly, reducing debugging time and improving code reliability.
